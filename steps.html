Creating a Custom Request Viewer with GitHub Pages

In this example we will take a look at a custom request that I made around Huntsville, Alabama. Let's use my custom request and create a website to share some findings with field staff, non-technical staff, and anyone interested.
and host it on your github!

The dates of this custom request where between the spring of 2010 and 2011.

Get Started

Step 1

Adding the HTML Files

Create a repository or repo.

Name it something meaninful.

Give it a license. By the way, MIT is a great choice.

add the Read me file. It's a good idea to put info about what the project is about and even better, instructions about how to use it.

Clone the repo to your desktop

Now lets create the HTML page, by downloading the file here: HTML Files
Then unzip the file into the cloned repo's base directory. This should be on your computer.

You now have a framework for a basic mapping web page hosted on githiub. It's missing some of the functionaility. Don't worry we will add a lot more with this shortly.

Use the GitHUb desktop to commit this code. Then "push" or sync it to GitHub.

Once this has completed use GitHub to create a branch named gh-pages. Goto the repo's setting and make it the default branch.
Now open a browser which is NOT Internet Explorer and go to the URL <your-GitHub-username>.github.io/<repo-name>

I like to add the url to the readme file so it's easy to navigate to the gh-pages site with a click. To do this edit the read me on the github webpage and copy and paste the url. Github markdown will automaically make it a link. Save and commit.

Go To Next Step

Step 6

Adding WMS layer toggle

So none of the toggle buttons for the wms layer do anything. Let's add an issue and work on the solution.

On GitHub add an issue wms layers not working.
Of course in real life a user somewhere in the world would do that part!

Then add a branch named: add wms layers. Note the issue number

Download the wms functionality here: wms files
unzip the file into the "js" directory.

Use GitHub desktop to commit and in the commit message include "this closes issue #<the acutal number assocaited with the issue>". Then "push" or sync the commit to GitHub.

Send a Pull Request to gh-pages.
If you like the code Accept the PR and then Merge and Delete the Branch.
Check out the changes.
